🌊 AlphaEarth 核心场景升级方案：江苏盐城红线审计 (V7.1 空间围栏靶向版)

一、 问题复盘与 V7.1 演进逻辑

前情提要：在 V7.0 中，我们成功引入了欧空局 ESA WorldCover 作为金标准，彻底解决了海岸线边界的“浑水误判”问题。但随之引发了新的地学问题：ESA 会将内陆所有的“城市/村庄”识别为建筑（红色），将内陆的“收割农田/工地”识别为裸土（黄色），导致画面重点失焦。

V7.1 核心解法：AI 特征对齐 + 空间地理围栏 (Geofencing)
“海岸线红线审计”是一个具有强空间属性的业务。违规围海造田只发生在地平线边缘。
因此，我们在后端的 GEE 算子层引入 Coastal Geofence (海岸带空间围栏)，使用多边形裁剪（Clip）技术，一刀切掉内陆市区的干扰，配合透明水体遮罩，让画面只剩下悬浮在海陆交界处的“红黄博弈带”。

二、 后端服务代码升级 (gee_service.py)

不需要重新训练 Asset 模型！我们只需要修改后端的渲染图层逻辑，加入精准的地理围栏裁剪。

请在 gee_service.py 中，找到 ch5_coastline_audit 的分支，替换为以下代码：

    elif ("ch5_coastline_audit" in mode_s) or ("海岸线" in mode_s) or ("红线审计" in mode_s):
        base_img = _select_embedding_bands(filtered_col.mosaic(), emb_bands)
        ASSET_ID = "projects/your-project/assets/classifiers/ch5_coastline_rf_v1" # 确保这里是您的真实资产ID
        classifier = ee.Classifier.load(ASSET_ID)
        
        # 使用基于 ESA 训练的 RF 模型进行推理 (0=水, 1=滩涂, 2=建筑围垦, 3=内陆林草)
        img = base_img.classify(classifier)
        
        # -----------------------------------------------------------------
        # V7.1 核心升级：增加海岸带空间地理围栏 (Coastal Geofence)
        # 物理隔绝内陆城市（如盐城市区）和内陆越冬裸土对“建筑”和“滩涂”分类的干扰
        # -----------------------------------------------------------------
        coastal_fence = ee.Geometry.Polygon([[
            [120.30, 34.00], # 西北边界 (避开市区)
            [121.50, 34.00], # 东北边界 (深海)
            [121.80, 32.50], # 东南边界 (深海)
            [120.60, 32.50], # 西南边界 (避开内陆农田)
        ]])
        
        # 1. 空间裁剪：只保留地理围栏内的图层
        img = img.clip(coastal_fence)
        
        # 2. 语义剥离：将内陆林草(3)和深海水域(0)完全透明化，只暴露红黄红线！
        img = img.updateMask(img.neq(3).And(img.neq(0)))
        
        vis = {
            "min": 0, 
            "max": 3, 
            "palette": [
                "000000",  # ID 0: 水域 (透明)
                "F6C431",  # ID 1: 自然泥沙沉积滩涂 (金黄色)
                "E23D28",  # ID 2: 人工围填海/硬化岸线/防波堤 (警告红)
                "000000"   # ID 3: 内陆背景 (透明)
            ]
        }
        suffix = "ch5_audit_geofence"
        return img, vis, suffix
