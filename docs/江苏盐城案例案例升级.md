AlphaEarth 核心场景升级方案：江苏盐城海岸线红线审计 (V6.6 终极版)

一、 业务痛点与技术路线重构

1. 为什么必须废弃 K-Means (动态聚类)？

在上一代系统中，我们使用 K-Means(A00, A02) 进行无监督聚类。这在实际业务（离任生态审计、35%自然岸线保有率计算）中暴露了三大致命缺陷：

标签漂移 (Label Switching)：K-Means 是无监督的，当视口改变时，原本属于“自然滩涂”的 Cluster 1 可能会被重新分配给“人工建筑”，导致前端出现“红海”等荒诞的渲染错误。

维度欠拟合：仅使用 A00 和 A02 两个维度，无法在长江口/黄海等复杂水域有效区分**“浑浊的泥沙水”与“真正固结的自然滩涂”**。

高并发计算灾难：每次前端拖拽地图，GEE 都要在云端重新采样、重新训练，极易引发 HTTP 504 超时。

2. 终极解决方案：零样本锚点 + 随机森林资产化 (Asset)

为了彻底解决上述问题，我们转向 “高维特征预训练 + 资产固化” 的终极路线：

引入多边形锚点 (Polygons)：在 AEF 16 维隐空间中，人工指定极具代表性的“特征基准面”（如：长江口极度浑浊区=水体，盐城条子泥=滩涂，南汇嘴防波堤=人工围垦）。

训练轻量级随机森林 (Random Forest)：让 AI 学习这些基准面的 16 维特征空间分布，强制模型理解“即便颜色发黄、充满泥沙，但具有流体波纹特征的依然是水”。

固化为 GEE Asset：将训练好的分类器打包成云端资产（Asset）。生产环境中直接加载该 Asset，实现 毫秒级推理，彻底解决高并发和标签漂移问题。

二、 步骤一：一次性离线训练与导出脚本 (Python)
请保证在./run_backend.sh中自动实现这一步骤
请在您的本地环境或 Jupyter Notebook 中运行一次以下脚本。该脚本会提取盐城及周边的典型地貌特征，训练分类器，并将其持久化到您的 Google Earth Engine (GEE) 云盘中。

前置准备：请确保已经通过 ee.Authenticate() 认证了您的账号。

import ee
import time

# 1. 初始化 GEE
ee.Initialize()

# 请替换为您自己的 GEE 项目 ID 或直接使用 user 路径
# 例如: "projects/your-project-id/assets/classifiers/ch5_coastline_rf_v1"
# 或者个人路径: "users/your-username/classifiers/ch5_coastline_rf_v1"
TARGET_ASSET_ID = "projects/your-project-id/assets/classifiers/ch5_coastline_rf_v1"

def export_classifier_to_asset():
    print("开始构建 AEF 海岸线生态分类器 (16维特征空间)...")
    
    # 获取 AEF 16维特征
    emb_col = ee.ImageCollection("GOOGLE/SATELLITE_EMBEDDING/V1/ANNUAL")
    emb_bands = [f"A{idx:02d}" for idx in range(0, 16)]
    # 使用近两年的影像进行训练，保证特征的鲁棒性
    base_img = emb_col.filterDate("2023-01-01", "2025-01-01").mosaic().select(emb_bands)
    
    # -----------------------------------------------------------------
    # 定义生产级基准样本区 (Polygons 包含多像素方差，抗噪性极强)
    # -----------------------------------------------------------------
    # Class 0: 水域 (强制模型将东海清水和长江口浑浊水统一识别为流体水域)
    water_clear = ee.Feature(ee.Geometry.Rectangle([122.3, 30.8, 122.6, 31.2]), {'class': 0})
    water_turbid = ee.Feature(ee.Geometry.Rectangle([121.8, 31.3, 122.0, 31.5]), {'class': 0}) 
    
    # Class 1: 自然滩涂 (固结的泥沙沉积带)
    mudflat = ee.Feature(ee.Geometry.Rectangle([120.8, 33.4, 121.0, 33.6]), {'class': 1})
    
    # Class 2: 人工硬化/围海造陆 (具有几何直线和高硬度特征)
    urban = ee.Feature(ee.Geometry.Rectangle([121.4, 31.1, 121.6, 31.3]), {'class': 2})       
    reclamation = ee.Feature(ee.Geometry.Rectangle([121.85, 30.85, 121.95, 30.95]), {'class': 2}) 
    
    training_regions = ee.FeatureCollection([water_clear, water_turbid, mudflat, urban, reclamation])
    
    print("正在 GEE 云端对基准多边形进行特征采样 (2000+ 像素)...")
    training_data = base_img.sampleRegions(
        collection=training_regions,
        properties=['class'],
        scale=30,
        tileScale=4
    )
    
    print("构建 Random Forest 模型计算图...")
    classifier = ee.Classifier.smileRandomForest(10).train(
        features=training_data, 
        classProperty='class', 
        inputProperties=emb_bands
    )
    
    print(f"准备导出分类器至: {TARGET_ASSET_ID}")
    task = ee.batch.Export.classifier.toAsset(
        classifier=classifier,
        description='Export_Coastline_RF_Classifier',
        assetId=TARGET_ASSET_ID
    )
    
    task.start()
    print("✅ 导出任务已成功提交至 GEE 后台！")
    print("请访问 [https://code.earthengine.google.com/tasks](https://code.earthengine.google.com/tasks) 查看导出进度。")
    print("导出完成后（通常需 1-3 分钟），即可在后端 API 中直接按 ID 加载该分类器。")

if __name__ == "__main__":
    export_classifier_to_asset()
