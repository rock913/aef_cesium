Zero2x v7.5：OneAstronomy 驱动的数字孪生宇宙与科学示范蓝图

文档代号：Phase 2.0 - "One Universe, Real Science"

一、 核心设计哲学：从“特效展示”走向“计算重塑”

基于对大规模数字巡天的深刻洞察，Zero2x v7.5 必须摒弃过去“各自为战、割裂加载”的特效 Demo 模式。我们将重塑三大核心理念：

One Universe（绝对连续的单一宇宙空间）：
抛弃“切换场景”的概念。所有的科学事件（CSST 数据涌入、星系分解、瞬变源爆发）均发生在一个以地球为视角的、具有对数级物理深度的统一 3D 笛卡尔坐标系中。所有的转场必须通过摄像机的空间穿梭（FlyTo）或视角缩放（ZoomIn）完成，实现“一镜到底”的沉浸感。

Micro-Real-Data（微型真实数据注入）：
摒弃前端的 Math.random()。我们将截取真实的公开观测数据（如 SDSS/Gaia 的万级样本切片），转为极简的 JSON 格式在前端加载。让用户肉眼看到真实的“宇宙网骨架（Cosmic Web）”，建立不可辩驳的科研公信力。

Workflow as Narrative（基于真实科研流的叙事）：
将零散的 Demo 编织为“数据获取 -> 模型特征提取 -> 物理演算 -> 发现新知”的连贯工作流，直接挂载 CSST、GOTTA 等国家级工程任务。

二、 模型底座与空间架构升级

2.1 OneAstronomy 天文科学基础模型

系统核心 AI 引擎正式定名为 OneAstronomy。它是一个内嵌物理一致性的多模态大模型。

模态特征对齐：支持图像（CSST测光图）、光谱（高低分辨率）、时域（光变曲线）和元数据的通用分词与特征融合。

Agent 工具簇映射：

decompose_csst_galaxy(image_tensor): 复杂星系精细结构分解

predict_cosmological_redshift(multimodal_data): 暗弱星系红移预测

capture_transient_event(time_series_data): 瞬变源秒级捕获与分类

2.2 WebGPU + 空间霸权渲染管线 (Scene Authority)

统一场景树：初始化时，构建一个庞大但扁平的 macroScene。

无界融合 (Additive Blending)：所有的二维面片（如星云特写）必须使用纯黑背景与加色混合（Additive Blending），并辅以边缘羽化遮罩（Vignette），消除任何“截图感”与“方形边界”。

层级降维：当聚焦某一特定事件时，背景星空执行“主动退让”（透明度降至 15% 甚至在 Z 轴上压平），凸显当前科学任务主体。

三、 全新连贯叙事剧本：科学示范四大任务

整个系统的交互将由左侧的 021 Copilot 自然语言驱动，引领用户在这个连续的宇宙中完成四项重大科学示范任务。

序幕：宇宙基底的唤醒 (The Data Fabric)

视觉呈现：从 Cesium 地球视界急速拉升至深空（一镜到底交接）。画面呈现一片寂静但真实的二维全天点云（基于真实巡天样本预加载）。

前端 UI：屏幕中央闪烁数据流加载指示器 [Loading CSST Sky Survey Data Array... 2.4 million sources]。

Copilot 旁白：“欢迎接入 Zero2x 天文数字孪生底座。当前展现的是基于测光数据的二维宇宙。我们将利用 OneAstronomy 模型，重新定义您的观测与数据处理流程。”

任务 1：CSST 复杂星系精细结构分解 (重塑数据处理)

业务痛点：应对 CSST 年均 1PB 数据，传统 Sersic 拟合耗时且存在主观偏差。

交互指令：“启动 CSST 数据流，对前方星系团进行精细结构分解。”

连续运镜：相机（Camera）平滑推进，聚焦到二维点云中的一个极其明亮的星系目标。

视觉演进：目标星系放大，OneAstronomy 模型下发指令。星系图像瞬间被“解剖”为不同颜色的图层——核球 (Bulge)、盘 (Disk)、棒 (Bar) 浮空分离展示。

价值传达：HUD 弹出看板：[解析完成。效率提升: 100倍。避免物理混淆！]

任务 2：精密宇宙学与红移立体爆裂 (领域科学家合作)

业务痛点：宇宙学模型样本不足，传统物理仿真（2500次宇宙推演）需 75万卡时。

交互指令：“与 Jean-Paul Kneib 团队协作，利用 OneAstronomy 预测暗弱星系红移，重构小尺度宇宙结构。”

连续运镜：相机迅速向后拉远（Zoom Out），回归宏观宇宙视角。

视觉演进（核心震撼点）：接收到模型下发的批量红移数据（$z$），原本平坦的百万个星系点云，瞬间爆发出耀眼的粉紫色光芒，并沿着相机的景深方向（世界 Y/Z 轴）以指数级向外拉伸膨胀，形成极其宏伟的“斯隆长城”三维立体网络。

价值传达：看板提示：[暗弱星系信噪比提升。可测数量提升: 4X。模型推演耗时: 4卡时 (加速 10^5 倍)]

任务 3：GOTTA 智能时域网络与瞬变源捕获 (重新定义观测)

业务痛点：早期暂现源信号微弱，捕捉瞬变源爆发的第一束光至关重要。

交互指令：“接入 GOTTA 巡天网络，全天候扫描异常时域波形。”

连续运镜：在宏伟的三维宇宙网中，相机的视角开始缓缓巡视。突然，深空极远处爆发出一个极其明亮的、闪烁的“像素点”（Supernova / TDE）。

视觉演进：系统触发“秒级捕获”警报！相机立刻执行 CatmullRom 样条曲线运镜，以“空间跃迁”的视觉效果极速逼近该闪烁点。OneAstronomy 模型介入，瞬间在星体旁展开时域光谱分析图（ECharts），完成真伪辨别与分类。

价值传达：展示时域闭环：[GOTTA 发现异常 -> OneAstronomy 目标识别分类 -> 启动高价值目标自主观测后随]

任务 4：模态互生与物理一致性微调 (基础模型能力兜底)

交互指令：“目标已被星际尘埃遮挡。利用 OneAstronomy 的多模态能力，反推其 X 射线辐射场。”

视觉演进：直接在当前的观测目标上执行。雷达扫描波纹掠过，原本因尘埃而暗淡的可见光图像（采用加色混合渲染），无缝褪变为剧烈辐射的 X 射线模拟图谱（高频 Bloom 噪点）。证明该大模型并非仅仅是分类器，而是真正理解物理分布的生成式引擎。

四、 研发实施规范与前后端契约

为了实现上述“单宇宙连续流”，开发团队需遵循以下执行契约：

绝对禁止路由跳转：上述四个任务必须在 ThreeTwin.vue 的同一个 WebGPURenderer 上下文中完成。通过 store.currentAgentAction 驱动状态机。

小数据池预置 (Micro-Dataset)：

抓取约 2MB-5MB 的真实天体三维坐标 JSON 预置于前端 public/data/astronomy/。

初始化时一次性存入 WebGPU 的 StorageBuffer。

Tween 运镜库封装：
将所有的摄像机移动封装为带有动画回调的函数。例如：diveToTarget(vector3, duration, onComplete)，确保动画结束前禁止响应下一次宏观事件。

Copilot 的叙事主导：
右侧的 021 Copilot 不再仅仅是打印冰冷的日志，而是作为**“科学解说员”**，在每一个镜头推进时，同步打印出 CSST 数据加载...、OneAstronomy 物理硬约束验证中... 等具有极强业务属性的进展文本。

总结：这一版重构彻底打通了《数字孪生落地指引》中的核心卖点。它不再是“看四个特效”，而是向潜在的科学家、航天机构和投资人展示：在 Zero2x 的同一个宇宙中，数据是如何源源不断地从 CSST 流入，如何被 OneAstronomy 进行十万倍加速的红移预测，又如何指挥 GOTTA 网络完成瞬时观测的。 这才是一个真正的、可落地的“计算型数字孪生工作台”。
